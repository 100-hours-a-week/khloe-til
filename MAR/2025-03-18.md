#### ERD
: 데이터 베이스 구조를 설계할 때 엔터티와 속성, 관계를 나타낸 다이어그램 표
- Crow’s Foot Notation (까마귀 발 표기법)
- Peter Chen Notation (피터 챈 표기법)
- Barker Notation (바커 표기법)


#### JOIN
: 2개 이상의 테이블의 공통된 컬럼들을 짜집기하여 조회하는 방법입니다.
- 가로(수평) 확장
- 조인 방식에 따라 결과가 다름(`LEFT JOIN`, `RIGHT JOIN`, `INNER JOIN`)

#### VIEW
: 빠른 조회와 재사용, 유지보수를 용이하게 하기 위해 사용합니다. 단, 단, 업데이트가 많은 경우는 오히려 비효율적입니다.
 → 업데이트가 자주 되는 테이블을 조인하기보다는 캐싱 또는 물리 테이블에 집계된 데이터를 주기적으로 업데이트하는 방식이 더 나을 수 있습니다.

#### UNION
: 2개 이상의 SELECT문을 연결하여 각 쿼리의 결과를 하나로 합칩니다.(열의 개수와 데이터 타입이 동일해야함)
- 세로(수직) 확장
- 중복 데이터 자동 제거 (중복 포함시에는 `UNION ALL`)

#### Index 
: 테이블에 대한 **조회 동작 속도**를 높여주는 자료 구조
- 클러스터형 인덱스: 인덱스가 순서대로 물리적으로 정렬되어 저장되는 인덱스
- 비클러스터형 인덱스: 물리적으로 분리된 데이터와 인덱스로 인덱스 테이블에 포인터만 가지고 있어 원본 데이터를 참조할 수 있게끔 함
동작 방식 : 생성 - 검색 최적화 - 유지 관리 - 삭제

#### Binary Search Tree(이진 탐색 트리,BST)
1. 각 노드에 값이 있다.
2. 값들은 정렬된 순서가 있다. (트리 안에서 어느 값이 더 크고 작은지 명확히 알 수 있습니다.)
3. 왼쪽 서브트리에는 그 노드의 값보다 작은 값들을 지닌 노드들로 이루어져 있다.
4. 오른쪽 서브트리에는 그 노드의 값보다 큰 값들을 지닌 노드들로 이루어져 있다.
5. 좌우 하위 트리는 각각이 다시 이진 탐색 트리여야 한다.


#### SQL Injection
  : 웹 에플리케이션에서 사용하는 데이터베이스에 악의적인 목적으로 SQL 쿼리문을 주입해 실행되게 하여 데이터베이스를 조작, 빼내는 공격 기법
